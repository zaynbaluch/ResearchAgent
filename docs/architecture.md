# Architecture Overview

The Multi-Agent Research Assistant is built using a modern decoupled architecture. It separates the real-time agent workflow (LangGraph/FastAPI) from the interactive client interface (Next.js/React).

## System Components

### 1. Backend (FastAPI & LangGraph)
The backend is a Python application that orchestrates multiple specialized agents using **LangGraph**. It serves a REST API via **FastAPI** with Server-Sent Events (SSE) to stream real-time execution states to the client.

- **State Management**: LangGraph maintains the conversation state across interactions. `aiosqlite` is used alongside LangGraph's `AsyncSqliteSaver` to persist conversation history (memory check-pointing), enabling multi-turn capabilities.
- **LLM Integration**: Uses `langchain-groq` for high-speed inference.
- **Search Tooling**: Integrates with the `Tavily` search API for robust, web-based business data retrieval.

#### Agent Pipeline
The workflow is a conditional directed graph with 4 distinct agents:
1. **Clarity Agent**: Assesses the user's initial query for specificity. If the query is ambiguous (e.g., "Tell me about that tech company"), it sets a `needs_clarification` flag. This triggers a Human-in-the-Loop (HITL) interrupt, halting the pipeline and asking the user for details before proceeding.
2. **Research Agent**: Activated once the query is clear. It utilizes the Tavily API to gather extensive information (news, financial figures, company background). It assigns a `confidence_score` (0-10) to its findings.
3. **Validator Agent**: Evaluates the research gathered. If the research confidence is low or information is insufficient, it triggers a **retry loop** back to the Research Agent (up to a set maximum number of attempts).
4. **Synthesis Agent**: Receives the validated research data and compiles it into a comprehensive, well-structured, Markdown-formatted report for the end user.

### 2. Frontend (Next.js & React 19)
The frontend is a dynamic Single Page Application (SPA) built with Next.js and React 19, focusing heavily on interactive UI and real-time feedback.

- **State Management (`zustand`)**: Centralized state management for chat history, active agent tracking, and system confidence scores.
- **Real-time Event Streaming**: The `ChatWindow` component uses the Fetch API to listen to Server-Sent Events (SSE) from the FastAPI backend. It streams Markdown tokens live and updates the status of the pipeline (active, complete) in real time.
- **Visualizations**:
  - **Agent Graph Panel**: Built with `@xyflow/react` (React Flow), this provides a real-time visual representation of the LangGraph pipeline, dynamically highlighting which agent is currently active, and revealing the conditional edges (like retries or bypasses).
  - **Confidence Timeline**: Uses `recharts` to plot the confidence scores generated by the Research Agent over time.
- **Styling**: Relies on modern Vanilla CSS with CSS Variables (`globals.css`) for consistent theming, dark-mode styling, and aesthetic micro-animations without external heavy utility frameworks.
